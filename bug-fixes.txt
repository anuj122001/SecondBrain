  by GPT-4o-mini with citations.

  Bug 2 — File upload always fails in Dashboard.jsx:41

  The upload request manually sets Content-Type: 'multipart/form-data' without
  the required boundary parameter. This breaks multer's ability to parse the
  incoming file — the field req.file will be undefined on the backend and every
  upload returns a 400 error.

  await axios.post('http://localhost:4000/api/documents/upload', formData, {
    headers: { 'Content-Type': 'multipart/form-data' }  // ← missing boundary
  });

  The fix is to remove the headers override entirely and let the browser set the
   correct Content-Type automatically for FormData.

  ---
  Bug 3 — user.email is undefined after page refresh (AuthContext.jsx:17)

  On a fresh page load with an existing token, AuthContext sets:
  setUser({ authenticated: true });  // no email field
  But Dashboard.jsx:89 renders {user.email}, which shows undefined. The email is
   only available when login/signup is called (not on token hydration).

  ---
  Bug 4 — Text file content is never extracted (processor.js:53)

  Any file that isn't a PDF or image gets a hardcoded mock string instead of its
   actual content:
  } else {
    rawText = "Mock text for non-PDF/Image files.";  // ← .txt files get this
  }
  Text files uploaded by users are effectively empty in the knowledge base.
